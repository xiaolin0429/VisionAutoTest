# Backend Docs

本目录存放 VisionAutoTest 后端相关的长期文档，面向后端研发、Tech Lead、代码审查和结构治理场景。

## 适合什么时候读

- 你要理解后端整体架构和边界
- 你要新增或修改后端接口、模型、服务、执行链路
- 你要核对报告、基准、认证、执行基础设施相关规则
- 你要做后端 review 或结构治理

## 推荐阅读顺序

1. [服务端技术架构设计文档](./服务端技术架构设计文档.md)
2. [认证生命周期规范](./认证生命周期规范.md)
3. [报告与证据模型规范](./报告与证据模型规范.md)
4. [基准维护闭环规范](./基准维护闭环规范.md)
5. [执行基础设施抽象规范](./执行基础设施抽象规范.md)

## 文档分工

- `服务端技术架构设计文档.md`
  说明后端总体架构、技术选型和核心链路。
- `认证生命周期规范.md`
  说明 token、session、错误码和前端联动语义。
- `报告与证据模型规范.md`
  说明报告摘要结构、失败原因码和证据产物分类。
- `基准维护闭环规范.md`
  说明模板基准版本、失败采纳和溯源规则。
- `执行基础设施抽象规范.md`
  说明 dispatcher、storage、artifact_url 等基础设施抽象。

## 当前维护原则

本目录只保留长期有效的后端架构与规范文档。

原则上不再长期保留以下类型文档：

- 已完成阶段的重构计划
- 一次性执行移交单
- 仅服务某一轮改造的专项 review 清单

这类内容如果还有长期约束价值，应优先沉淀到：

- `backend/README.md`
- `.claude/skills/visionautotest-backend-standard-flow/SKILL.md`
