# 执行基础设施抽象规范

## 1. 调度抽象（Dispatcher Abstraction）

### 1.1 当前实现

**接口定义**
```python
class TestRunDispatcher(Protocol):
    def dispatch_test_run(self, test_run_id: int) -> None:
        """异步调度测试执行批次"""
        ...
```

**MVP 实现：InProcessTestRunDispatcher**
- 基于 FastAPI BackgroundTasks
- 进程内异步执行
- 适用于单机部署

### 1.2 扩展路径

**企业级实现：CeleryTestRunDispatcher（待实现）**
- 基于 Celery + Redis
- 分布式任务队列
- 支持多 Worker 并发
- 适用于集群部署

### 1.3 接口稳定性保证

- 调度接口保持单一方法：`dispatch_test_run(test_run_id)`
- 不暴露底层实现细节（BackgroundTasks/Celery）
- 通过依赖注入切换实现：`get_test_run_dispatcher()`
- 执行逻辑（process_test_run）与调度机制解耦

## 2. 存储抽象（Storage Abstraction）

### 2.1 当前实现

**接口定义**
```python
class StorageBackend(Protocol):
    def store_bytes(self, *, namespace: str, file_name: str, content_bytes: bytes) -> StoredObject:
        """存储字节流，返回对象键"""
        ...

    def resolve_path(self, object_key: str) -> Path:
        """解析对象键为本地路径"""
        ...
```

**MVP 实现：LocalStorageBackend**
- 基于本地文件系统
- 对象键格式：`{namespace}/{token}_{filename}`
- 适用于单机部署和开发环境

### 2.2 扩展路径

**企业级实现：S3StorageBackend（待实现）**
- 基于 S3 协议（MinIO/OSS/AWS S3）
- 支持分布式对象存储
- 支持 CDN 加速
- 适用于生产环境

### 2.3 接口稳定性保证

- 存储接口保持两个核心方法
- 不暴露底层存储细节（文件系统/S3）
- 通过依赖注入切换实现：`get_storage_backend()`
- 对象键作为统一标识符，不依赖具体存储实现

## 3. 报告产物访问模型

### 3.1 当前设计

**数据模型**
```python
MediaObject:
  - object_key: str  # 存储层标识符
  - file_name: str
  - mime_type: str

ReportArtifact:
  - media_object_id: int
  - artifact_url: str  # 统一访问路径
  - artifact_type: str
```

**访问路径**
- 统一格式：`/api/v1/media-objects/{media_object_id}/content`
- 不暴露底层 object_key
- 前端只需要 artifact_url，无需关心存储实现

### 3.2 面向未来兼容性

**当前（LocalStorage）**
```
artifact_url → /api/v1/media-objects/7001/content
           ↓
API 查询 MediaObject.object_key
           ↓
LocalStorageBackend.resolve_path(object_key)
           ↓
返回本地文件
```

**未来（S3Storage）**
```
artifact_url → /api/v1/media-objects/7001/content
           ↓
API 查询 MediaObject.object_key
           ↓
S3StorageBackend.generate_presigned_url(object_key)
           ↓
重定向到 S3 预签名 URL
```

### 3.3 兼容性保证

- artifact_url 格式保持稳定
- 前端无需修改代码
- 通过后端切换存储实现
- 支持透明升级到对象存储

### 4.3 切换示例

```python
# config.py
storage_backend_type: str = "local"  # local | s3
dispatcher_type: str = "inprocess"  # inprocess | celery

# storage.py
def get_storage_backend() -> StorageBackend:
    if settings.storage_backend_type == "s3":
        return S3StorageBackend(...)
    return LocalStorageBackend(...)

# dispatcher.py
def get_test_run_dispatcher(...) -> TestRunDispatcher:
    if settings.dispatcher_type == "celery":
        return CeleryTestRunDispatcher(...)
    return InProcessTestRunDispatcher(...)
```
