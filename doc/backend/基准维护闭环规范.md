# 基准维护闭环规范

## 1. 业务关系

### 1.1 核心实体关系

```
Template (模板)
  ├─ current_baseline_revision_id → BaselineRevision (当前基准版本)
  └─ BaselineRevisions[] (历史基准版本)
       ├─ source_type: manual/uploaded/adopted_from_failure
       └─ 如果 source_type = adopted_from_failure:
            ├─ source_report_id → RunReport
            ├─ source_case_run_id → TestCaseRun
            └─ source_step_result_id → StepResult
                 └─ actual_media_object_id → MediaObject (失败截图)
```

### 1.2 基准采纳流程

1. 用户在报告中查看失败的步骤结果
2. 确认失败截图符合预期（业务变更导致的合理差异）
3. 将失败截图采纳为新的基准版本
4. 系统创建新的 BaselineRevision，source_type = `adopted_from_failure`
5. 更新 Template.current_baseline_revision_id
6. 后续执行使用新基准进行匹配

## 2. 接口设计

### 2.1 现有接口（已实现）

**创建基准版本**
- 方法：`POST`
- 路径：`/api/v1/templates/{template_id}/baseline-revisions`
- 请求体：
```json
{
  "media_object_id": 7002,
  "source_type": "adopted_from_failure",
  "source_report_id": 8001,
  "source_case_run_id": 6002,
  "source_step_result_id": 9102,
  "remark": "adopt latest accepted screenshot",
  "is_current": true
}
```

**查询基准版本列表**
- 方法：`GET`
- 路径：`/api/v1/templates/{template_id}/baseline-revisions`

### 2.2 推荐新增接口

**从失败步骤采纳基准（便捷接口）**
- 方法：`POST`
- 路径：`/api/v1/step-results/{step_result_id}/adopt-as-baseline`
- 请求体：
```json
{
  "template_id": 5001,
  "remark": "业务变更：新增顶部横幅",
  "set_as_current": true
}
```
- 响应：返回新创建的 BaselineRevision

说明：
- 自动从 step_result 提取 actual_media_object_id
- 自动关联 report_id, case_run_id, step_result_id
- source_type 固定为 `adopted_from_failure`
- 简化前端调用流程

## 3. 业务规则

### 3.1 采纳前置条件

- 步骤结果必须存在 actual_media_object_id
- 步骤类型必须是 template_assert
- 步骤状态必须是 failed（断言失败）
- 目标模板必须存在且状态为 published
- 用户必须有目标模板所在工作空间的写权限

### 3.2 采纳后行为

- 创建新的 BaselineRevision，revision_no 自动递增
- 如果 set_as_current = true，更新 Template.current_baseline_revision_id
- 同一模板只能有一个 is_current = true 的版本
- 历史版本保留，支持回滚

### 3.3 溯源能力

- 每个 adopted_from_failure 类型的基准版本都可追溯到：
  - 哪次执行批次（report_id）
  - 哪个用例实例（case_run_id）
  - 哪个步骤结果（step_result_id）
- 前端可展示"此基准来自 2026-03-29 的执行 #8001"

## 4. 前端集成指南

### 4.1 失败步骤展示

```typescript
// 在步骤结果详情页，如果是 template_assert 失败
if (stepResult.step_type === 'template_assert' && stepResult.status === 'failed') {
  // 显示"采纳为新基准"按钮
  showAdoptButton({
    stepResultId: stepResult.id,
    templateId: stepResult.template_id, // 从步骤配置获取
    actualImageUrl: stepResult.actual_media_object_id
  })
}
```

### 4.2 采纳操作

```typescript
// 方式1：使用现有接口（需要手动提取信息）
await createBaselineRevision(templateId, {
  media_object_id: stepResult.actual_media_object_id,
  source_type: 'adopted_from_failure',
  source_report_id: reportId,
  source_case_run_id: caseRunId,
  source_step_result_id: stepResultId,
  remark: userInput,
  is_current: true
})

// 方式2：使用便捷接口（推荐，待实现）
await adoptStepResultAsBaseline(stepResultId, {
  template_id: templateId,
  remark: userInput,
  set_as_current: true
})
```

### 4.3 基准历史展示

```typescript
// 查询模板的基准版本历史
const revisions = await listBaselineRevisions(templateId)

// 展示每个版本的来源
revisions.forEach(revision => {
  if (revision.source_type === 'adopted_from_failure') {
    displaySource({
      type: '从失败采纳',
      reportId: revision.source_report_id,
      caseRunId: revision.source_case_run_id,
      stepResultId: revision.source_step_result_id,
      isCurrent: revision.is_current
    })
  }
})
```

## 6. 错误码

| 错误码 | 说明 |
|---|---|
| `STEP_RESULT_NOT_FOUND` | 步骤结果不存在 |
| `STEP_RESULT_NO_ACTUAL_IMAGE` | 步骤结果没有实际截图 |
| `STEP_TYPE_NOT_TEMPLATE_ASSERT` | 步骤类型不是模板断言 |
| `TEMPLATE_NOT_FOUND` | 模板不存在 |
| `TEMPLATE_NOT_PUBLISHED` | 模板未发布 |
| `BASELINE_REVISION_CONFLICT` | 基准版本冲突 |
