# 认证生命周期规范

## 1. Token 生命周期

| Token 类型 | 有效期 | 说明 |
|---|---|---|
| access_token | 7200秒 (2小时) | JWT格式，用于API认证 |
| refresh_token | 604800秒 (7天) | 用于续期access_token |

## 2. Refresh Token 状态机

```
active → used (续期成功)
active → expired (超过有效期)
active → revoked (会话吊销)
```

- `active`: 可用于续期
- `used`: 已完成一次续期，不可再用
- `expired`: 已超过有效期
- `revoked`: 会话吊销导致失效

## 3. 错误码规范

| 错误码 | HTTP状态 | 说明 | 前端处理 |
|---|---|---|---|
| `INVALID_CREDENTIALS` | 401 | 用户名或密码错误 | 提示用户 |
| `USER_DISABLED` | 403 | 账号已禁用 | 提示用户 |
| `TOKEN_EXPIRED` | 401 | 访问令牌过期 | 自动续期 |
| `TOKEN_REVOKED` | 401 | 访问令牌已失效 | 回到登录页 |
| `REFRESH_TOKEN_INVALID` | 401 | 刷新令牌非法或已消费 | 回到登录页 |
| `REFRESH_TOKEN_EXPIRED` | 401 | 刷新令牌已过期 | 回到登录页 |
| `REFRESH_TOKEN_REVOKED` | 401 | 刷新令牌已被吊销 | 回到登录页 |
| `SESSION_NOT_FOUND` | 404 | 会话不存在 | 回到登录页 |

## 4. 会话吊销语义

调用 `DELETE /api/v1/sessions/current` 时：

1. 会话状态设为 `revoked`
2. 所有关联的 `active` refresh_token 设为 `revoked`
3. 当前 access_token 立即失效（通过 jti 校验）

## 5. 前端联动

前端已对接以下错误码，触发时清理本地会话并回到登录页：

- `REFRESH_TOKEN_EXPIRED`
- `REFRESH_TOKEN_REVOKED`
- `REFRESH_TOKEN_INVALID`
- `TOKEN_REVOKED`
- `SESSION_NOT_FOUND`
