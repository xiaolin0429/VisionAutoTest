# 企业级跨端视觉自动化测试平台PRD

# 一、文档概述

## 1.1 文档目的

本文档为「企业级跨端视觉自动化测试平台」的产品需求文档，明确平台的核心目标、功能范围、技术架构、交互逻辑、非功能需求、开源&商用要求，用于指导开发过程（含AI辅助生成），确保平台可落地、可商用、可开源。该平台贴合测试工程师（包含功能、接口、UI、APP专项）的实际工作痛点，聚焦“维护易、鲁棒性高、低介入”的核心诉求。

## 1.2 核心目标

- 核心目标1：搭建一套基于成熟生态的视觉自动化测试引擎，以 Playwright 控制端 + OpenCV (多维比对) + OCR (文字认知) 为核心，从根本上解决传统 DOM/控件定位不稳定、一次 UI 改版导致全局脚本失效的痛点。
- 核心目标2：支持全端测试（PC/Mac Web、移动 H5、Android/iOS 端），通过自适应算法实现抗分辨率差异、抗设备缩放、抗轻微 UI 偏移等高维鲁棒性。
- 核心目标3：打造开箱即用的企业级测试管理台，包含数据驱动配置、模板动态基准管理（Baseline）、局部智能屏蔽（Ignore Regions）、任务调度、图文详尽报告等功能，适用于全员回归测试及业务验收场景。
- 核心目标4：保证系统具备弹性的架构支撑。支持轻量化本地单机运行，同时亦支持向企业级高并发及微服务架构平滑扩展，赋能测试工程师转型为高级测试开发 (TestDev)。
- 核心目标5：架构及选型高度兼容当前 AI Agent/大语言模型生成代码的最佳实践。通过严谨的接口契约与框架选取，降低重度代码编写门槛。

## 1.3 适用范围

本文档适用于研发工程师、测试开发工程师 (TestDev)、AI开发助手、产品维护者以及希望进行二次开发的开源贡献者。覆盖平台需求定义、架构选型、功能落地的全业务链路描述。

## 1.4 术语定义

- 视觉自动化测试 (Visual UI Testing)：摒弃元素路径强依赖（不依赖 XPath/Selector/ID），基于人类实际观察视角的图像识别、特征比对、OCR 认知，对界面内容进行像素级或特征级断言的高级测试手段。
- 动态基准维护 (Baseline Update)：发生正常 UI 迭代更新时，平台支持在测试报告页“一键采纳”新的快照进入模板库，替代传统的重复抓包手动更新操作。
- 局部忽略区域 (Ignore Regions/Masking)：通过在模板配置页画框标注，在比对时自动忽略经常变更的动态元素（如轮播广告、时间戳、用户头像），防范误报警。
- 特征点匹配 (Feature Matching)：基于 ORB/SIFT/SURF 等特征提取算法，实现抗形变、抗旋转、抗放缩等高鲁棒性识别。
- 环境沙箱隔离 (Context Isolation)：自动化每次执行皆启动干净、无历史缓存、无脏 Cookie 的独立容器上下文，保证测试用例独立无污染。

## 1.5 配套实施文档

为避免单一文档过长导致实现歧义，项目已补充拆分式实施文档：

- API 详细契约以 `doc/api/` 下分模块文档为准；
- 数据库物理表设计以 `doc/database/` 下分模块文档为准；
- 项目交付边界与实施顺序以 `doc/MVP定义.md` 和 `doc/实施路线图.md` 为准。

# 二、产品定位与核心价值

## 2.1 产品定位

一款面向研发、测试及 DevOps 团队的高级（企业级）视觉自动化测试平台。基于现代 Python 栈（异步、易运维、易 AI 生成）及渐进式前端技术开发。聚焦降低维护成本、强化断言鲁棒性、实现真正的“一次编排、随处断言”，打破海外高昂商业工具（如 Applitools）的垄断壁垒。

## 2.2 核心价值

### 2.2.1 对个人（Dev/QA）
- **摆脱脚本维护地狱**：应对高频次的前端发版，再不用盯着控制台查是哪个元素的 class name 变了。
- **简历杀手锏**：通过部署与二次开发此类带有“图像识别原理”“高可用并发调度”“全栈数据流转”等标签的项目，快速完成从“手动点点点”或“普通自动化”向高级测试开发(TestDev)的职级突破。

### 2.2.2 对企业与团队
- **开源免费，即插即用**：提供足以媲美商用 SaaS 的视觉测试能力且完全支持内网私有部署，消除敏感数据上云的合规风险。
- **DevOps/CI 无缝整合**：为持续集成提供可靠的前端渲染验证关卡。将因“CSS样式错乱、组件层级覆盖”引发的线上回归大Bug拦截在发布前。

# 三、用户画像与使用场景

## 3.1 核心用户与画像

|用户类型|核心需求|操作预期|
|---|---|---|
|高级测试工程师 / 测试开发|建设体系化回归测试用例，实现无人值守每日巡检；需要易用的配置台以便将维护下放。|深度配置参数（特征点阈值、比对策略），建设数据驱动流程，编写扩展插件。|
|功能测试 / QA成员|每次前端迭代后快速验证相关页面 UI 显示否正确无重叠，确认文案一致性。|使用前端界面新建任务，圈选“忽略区域”，查看失败报告并一键采纳合理基境。|
|研发 / 前序基建平台|DevOps 流水线触发回归测试卡口，要求高并发快速出报告反馈阻断部署。|通过 RESTful Webhook 调用 API；查收结果推送及失败报表。|

## 3.2 典型使用场景

1. **多端多屏横纵向跨端适配测试**：研发交付新响应式站点，测试平台分别在 iPhone SE (小屏)、iPad (中屏)、4K 显示器 (大屏) 视窗下启动任务驱动引擎，通过 OCR 或特征比对验证核心按钮是否存在且未被遮挡。
2. **重度改版“基准一键升级”**：APP 全新大版本发布（主色调变化），原先旧用例大面积飘红失败。QA 登录平台，在最新报告中批量点击“确认基准变动合法，覆盖旧模板”，1 分钟内完成所有新版本用例自动化升级。
3. **动态乱序长列表页验真**：页面含有实时动态热榜新闻。QA 采用“局部忽略区域”工具将新闻标题内容区涂黑遮挡。平台运行时仅断言列表排版布局框架、底部栏和 Header 符合标准，忽略动态内容。
4. **验证非标准 UI 组件体系（Canvas / WebGL / 数据大屏）**：针对传统 DOM 的 WebDriver 完全提取不到标签图表库或游戏化交互页面，通过配置精准的模板坐标加阀值控制，成功实施测试验证。

# 四、技术架构设计需求（兼容高弹性与AI自动生成）

## 4.1 核心分层架构

采用演进式分层架构，既可以本地轻量级（Standalone）闭环运行，又能根据业务规模拓展为微服务集群形态。对于 AI 工具，这也是模块切分最清晰的标杆架构：

1. **前端交互展示层 (Frontend)**：建议采用 Vue3 + Tailwind CSS 的轻量级 / 渐进式前端方案。相比于纯原生 HTML+JS，该方案具备出色的数据双向绑定和组件化能力，能极为轻松地实现“画框涂抹忽略区域”、“拖拽编排用例”、“局部截图动态高亮”等强交互体验；且主流 AI 对此栈代码生成精准度极高。
2. **API 路由控制层 (API Gateway)**：基于 Python `FastAPI` 框架。使用异步通信处理高并发访问，自带完整的文档驱动（Swagger UI），提供与前端及外部 CI 系统交互的标准 REST 规范化服务接口。
3. **测试任务调度层 (Job Scheduler)**：引入任务队列机制。MVP 阶段默认采用单节点进程内异步任务机制，降低部署门槛；在企业级场景下，推荐升级为 `Celery + Redis` 以管理大量并发的 UI 测试请求，防止资源跑满导致的崩溃；定时任务可兼容 `APScheduler`。
4. **核心执行与推断引擎 (Core Enginge)**：
    * **驱动控制器**：采用 Python 版 Playwright，负责分配沙盒隔离（Context）、页面仿真、录屏截图与输入交互。
    * **计算机视觉比对**：封装 OpenCV (模板阈值 / ORB, SIFT 特征分析组合策略)。
    * **光学字符识别**：封装 PaddleOCR 等 AI 轻量化文字位置和内容推测能力。
5. **持久化数据与存储层 (Data Layer)**：
    * 关系型数据：使用 SQLAlchemy 作为 ORM 抽象代理。平台在 MVP 阶段建议默认采用 PostgreSQL 16 单实例；本地演示可退化到 SQLite；企业现网若已有 MySQL 8，也应支持兼容适配，从根本上解决系统高并发写入时的读写瓶颈。
    * 文件存储系统：提供本地或 S3 兼容的云端接入能力供存储图片、报告录像。
6. **环境基建侧部署层 (Infrastructure)**：全部件 Docker 容器化打包，提供 `docker-compose.yml` 实现零配置组网及网络互通。

## 4.2 架构选型矩阵汇总

| 分层维度 | 技术选型推荐 | 核心设计初衷与优势 |
|---|---|---|
| 前端展现层 | Vue3 + Tailwind CSS | 低门槛实现画框、拖拽等复杂交互。对AI生代码极为友好。 |
| API 接口层 | Python + FastAPI | 享用原生异步能力，强制契约生成，提供稳定高效网关抽象。 |
| 执行推理引擎 | Playwright + OpenCV + PaddleOCR | 当下自动化测试界生态主流，无痕沙盒技术有效防止上下文串扰。 |
| 任务分发列队 | Celery + Redis (可选内置) | 分离 HTTP 请求层和密集 IO/CPU 层测试消耗，防止架构雪崩。 |
| 数据持久与存管| SQLAlchemy + PostgreSQL/MySQL（SQLite 仅本地演示） | MVP 优先 PostgreSQL 16，企业级支持 MySQL 8，兼顾易用性与并发能力。 |
| 容器与分发环境| Docker + Docker Compose | 提供统一、极简的运行一致性容器，支持横向扩展 Worker 节点。|

# 五、核心功能模块需求细化（含企业级杀手功能）

## 5.1 数据模型与后端基建大盘

- **多租户/项目空间隔离 (Workspace / Project)**：企业级核心壁垒。打破扁平化数据池，引入“项目空间”概念。研发A部与电商B部的数据（用例、模板、执行记录、并发队列）实现高度软隔离，严防成员误操作他组核心资产。
- **环境与全局常量配置中心 (Environment Profiles)**：支持预设多套环境配置档案（如 `DEV`、`FAT`、`UAT`、`PROD`）。将测试 URL、初始高管账号、鉴权 Token 统一抽离管控。支持用例级和任务级的环境一键无缝切换，彻底告别“硬编码”。
- **RBAC权限体系**：基础管理员与平台普通用户的权限管控边界；保障越权操作被拦截校验。
- **环境预置库设定**：全局保存各类分辨率设备尺寸模型（iPhone 14 / Desktop 1080P等），网络模拟模板（4G / 断网等），上下文状态设定档。
- **数据监控与报表统计盘**：直观展示近30天内拦截缺陷趋势、自动化脚本健康度走势等聚合指标，辅助度量体系。

## 5.2 核心管理台业务侧（全端配置化）

### 5.2.1 模板与基准资产管理 (Template Asset Store)
- **多模态特征模板上传**：上传包含关键组件或全局截图的参考物库；平台需针对不同特征配置容忍阈值（0到1之间浮动）。
- **局部干扰区蒙版绘制 (Masking Regions) [高阶]**：通过前端拖拽交互产生 `X,Y,W,H` 相对座标数组，在 OpenCV 执行匹配时以遮罩将该区域做透明色谱过滤处理，实现高度抗噪。
- **动态基准合并升级 (Merge Baseline) [高阶]**：用例在正常版本更新执行失败后，支持点击“更新预期图”，新生成的失败现场截图瞬间被赋予为下一次判定周期的 Benchmark（基准图）。

### 5.2.2 高级执行流编辑与数据驱动测试引擎 (DDT)
- **公共断言组件与步骤复用 (Component Multiplexing)**：将高频测试链路（例如“标准账密登录”、“实名认证协议勾选”）封装为独立[公共组件]。主业务用例通过类似 `Call Component` 的方式直接调用，当公用链路视觉变动时，“改一处，全盘生效”，终结噩梦级的线性重构脚本痛点。
- **参数池与数据驱动支持 (Data-Driven)**：测试步骤中的输入值支持通过占位符表示法提取外部环境变量（如 `{{ test_username }}`），进而支持上传 CSV 多行数据集并发遍历同一视觉操作链条。
- **执行环境清理自决**：每一项包含独立设定的选项卡——【测试前清理】：是否开启无痕、是否清除远层端缓存或特定服务端数据接口的清理 Mock。
- **顺序拖拽编排机制**：提供界面允许组合使用（断言视觉块A -> 点击块A座标 -> 填入文字 -> 断言文字回显OCR）。

### 5.2.3 异常排查流与多维度全系测试报告系统
- **多渠道预警与触达机制 (Alerting & Webhook)**：提供全方面的信息触达网络，支持将执行失败率异常报警或 Nightly-Build 最终报告自动推送到团队常用工作台（飞书、钉钉、企业微信机器人、Slack）。卡片消息直观展示拦截数、报错节点截图热图。
- **高保真可追溯报表**：提供完整的时序线，精确映射步骤树。点击每个出错节点，自动以弹窗并列呈现：“预期特征图基准” vs “引擎实时寻道高亮图” vs “相似度差异热力图”。
- **崩溃原因智能定性提示**：根据日志自动对失败节点归档分类（如：”特征值不足(可能 UI 变化)“、”超时未能渲染 DOM (可能网络拥塞)“、”被其它浮窗阻挡 (弹窗未关闭)“），极大地消灭人工核对时间。

## 5.3 底层计算机视觉与跨端自动化引擎核心能力

### 5.3.1 多策略视觉防碎断言体系 (Robust Assertion)
- **自动分辨率容错补偿**：当源模板是从 Retina 双倍屏生成，而在标准库渲染执行时，内部框架执行自动 Resize 计算平滑化插值后再次对比。
- **抗形变/抗偏移特性点群发识别**：不仅依靠暴力模板，结合 SIFT/ORB 计算获取两图核心关键特征聚类群的 Homography 映射单应性矩阵变换，推导最终位置。
- **智能文本挖掘验证 (PaddleOCR)**：能对画布内的动态验证文字是否存续或内容变化做断言验证，解决动态订单号、流水号匹配难题。

### 5.3.2 分发跨终端驱动控制网(Playwright)
- 支持单进程生成多开 Browser Instances 并发打满服务器多核。配合无头运行降低 GPU 渲染功耗。
- 通过 `--device` 参数化无缝投屏至模拟移动设备渲染引擎；
- 对于全链路中长等待做重试和 Poll (轮询) 处理机制（默认每 0.5s 高刷新帧试图比对一次并判断成功，超时15秒抛出）。

## 5.4 系统运维调度、监控台与 API 规范化

- **执行节点与计算大盘监控 (Worker Monitoring Dashboard)**：为架构师/管理员提供对下挂载的所有 Celery Worker 和 Playwright 无头实例运行负荷监控可视化页面。可随时查看当前活跃实例数、空闲卡牌数及队列阻塞任务积压（Queue Depth），支持一键强杀僵死任务以释放宿主计算资源。
- **完备的 Swagger/OpenAPI 定义**：接口设计严格遵守 RESTful 规范。对于执行触发场景，统一通过 `POST /api/v1/test-runs` 创建异步执行资源，再通过 `GET /api/v1/test-runs/{test_run_id}` 或关联报告资源查询状态与结果，避免使用动作型路径。
- **任务定期兜底清理策略（Cron）**：定时表达式解析系统定期将系统的老旧截图与废弃记录自动清空丢弃，防止图文膨胀拖垮服务器磁盘存储大盘。

# 六、非功能与平台级落地保障约束

## 6.1 性能与并发度支撑能力
- 前端管理系统静态资源加载及首屏白屏时间 < 1.5 秒。响应丝滑，无厚重交互交互堵塞。
- 数据与引擎解耦，在企业规模下，配置 4核 8G 的 Workers 集群能平滑承接 20+ 个高频长时多端断言会话而不崩溃宕机，且不存在数据库 `Table Locked`。

## 6.2 代码美感及平台拓展性
- 全栈代码遵照 Python `PEP8`，FastAPI Pydantic 原生模型数据严格验证拦截清洗请求。保证不产生 SQL 注入与脏乱乱投射。
- 所有外置扩展接插件抽象为基类 `class EngineStrategy(ABC)` 和 `class StorageDriver(ABC)`。未来切换 OCR 引擎或加入图像生成大模型对比时只需要填鸭实现对应继承方法。

## 6.3 稳定性与安全防御边界
- 执行测试代码逻辑采用 Sandbox 进行约束隔离，所有对外访问的 HTTP Client 使用强超时期，杜绝由于被测业务服务无响应带来的挂卡死僵死平台自身。
- 内核支持断点重试与故障逃逸；当遇见执行 Worker 被宿主 OOM Kill 时能由队列拉起二次补偿重试。

# 七、落地实施、AI并行交付标准作业指导

为确保该方案在使用现代 AI 工具构建与开发时的高成功率，建立该交付指导。

## 7.1 开发策略建议 (AI Prompt 最佳分解范式)

**切忌使用一通到底的提示词要求 AI 生成完整项目，强烈推荐采用分层、分治递进式流水线**。
1. **[步骤 一：契约层]** 率先引导 AI 根据本 PRD 生成数据库实体类 (SQLAlchemy Models) 以及 FastAPI 的路由及基于 Pydantic 的出入参 Request/Response 定义，制定规范。
2. **[步骤 二：核心层]** 限定条件引导 AI 编写 `VisualMatchingEngine` (封装 OpenCV/Playwright 的类)。构建严谨参数且配有 Pytest 单元测试集，调试通过后再集成。
3. **[步骤 三：任务层]** 引导 AI 编写 Celery 任务或后台多线程去组装核心层的逻辑并入库流转更新。
4. **[步骤 四：视图层]** 在明确 API 的前提下，引导 AI 通过 Vue3/Tailwind 组装具有极强现代感的数据报表与交互管控台页面。

## 7.2 整体开发交付产物与交付基准

- **全库脚手架包**：带有按最佳实践划分项目目录的 (`app/api`, `app/core/vision`, `app/models`, `frontend/`) 分层源码体系。
- **环境基石配置文件**：`docker-compose.yml` (打包所有微服务组件)、`requirements.txt`。并支持跨 Linux/Mac OS/Windows 全境本地即插即点运行 (`python main.py` 极简式退化运行模式保障）。
- **极简的操作部署手册**：内聚 5 分钟上手视频图文流程。说明各类容差特性在平台中的应用设置经验指导原则。
