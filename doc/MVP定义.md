# MVP 定义

## 1. 目标

MVP 阶段只解决“最小可用的视觉自动化闭环”，不追求一次性实现完整企业级平台。目标是让团队能在单机或单容器环境中完成以下闭环：

- 登录进入工作空间。
- 配置测试环境和设备预设。
- 上传模板图并圈选忽略区域。
- 编排测试用例和测试套件。
- 触发一次异步执行。
- 查看执行结果、差异图和基础报告。

## 2. 纳入 MVP 的能力

### 2.1 后端

- 基于 `FastAPI` 的 RESTful API。
- 单机数据库模式。
- 单节点异步任务执行。
- 基准图与媒体文件本地存储。
- Playwright 基础页面驱动。
- OpenCV 模板匹配与忽略区域处理。
- 执行结果与报告查询。

### 2.2 前端

- 登录页。
- 工作空间切换。
- 环境配置页。
- 模板管理页。
- 用例编辑页。
- 套件管理页。
- 执行记录页。
- 报告详情页。

## 3. 暂不纳入 MVP 的能力

- 分布式 Celery Worker 集群。
- 多对象存储适配。
- 完整 RBAC 细粒度权限矩阵。
- 多渠道告警编排。
- 智能失败归因。
- 大规模监控大盘。
- 复杂审批流和基准批量合并。

## 4. MVP 验收标准

- 单次执行链路从触发到报告可完整跑通。
- 至少支持 1 种模板匹配策略和 1 种 OCR 断言策略。
- 忽略区域在不同分辨率下可按相对坐标生效。
- API 文档和数据库设计与实现保持一致。
- 有最小级别的单元测试和集成测试。

## 5. MVP 技术基线

- 后端：Python 3.11、FastAPI、SQLAlchemy 2.x、Alembic。
- 前端：Vue 3、Vite、Pinia、Tailwind CSS。
- 视觉引擎：Playwright、OpenCV、PaddleOCR。
- 数据库：优先 PostgreSQL 16；若企业已有 MySQL 8，可做兼容适配。
