# 接口示例与错误处理

## 1. 后端初始化示例

### 1.1 本地开发初始化

```bash
cd backend
python -m app.db.bootstrap
```

预期效果：

- 若 `AutoTestDev` 不存在则自动创建
- 自动执行 Alembic 迁移
- 自动写入默认管理员

### 1.2 启动后端

```bash
uvicorn app.main:app --reload
```

### 1.3 健康检查

```http
GET /healthz
```

成功示例：

```json
{
  "data": {
    "status": "ok",
    "service": "VisionAutoTest Backend"
  },
  "meta": {
    "request_id": "req_xxx"
  }
}
```

## 3. 创建组件并发布

```http
POST /api/v1/components
Authorization: Bearer <token>
X-Workspace-Id: 1
Content-Type: application/json

{
  "component_code": "shared_login",
  "component_name": "共享登录组件",
  "status": "published",
  "description": "登录页公共操作"
}
```

响应要点：

- `status` 只允许 `draft / published / archived`
- 首次发布时会返回 `published_at`

## 5. 创建执行批次

```http
POST /api/v1/test-runs
Authorization: Bearer <token>
X-Workspace-Id: 1
Idempotency-Key: run-001
Content-Type: application/json

{
  "test_suite_id": 3001,
  "environment_profile_id": 4001,
  "trigger_source": "manual"
}
```

成功响应要点：

- `status` 初始为 `queued`
- 后续通过 `GET /api/v1/test-runs/{id}` 轮询
- 数据库切换为 PostgreSQL 后，该接口协议未发生变化

## 6. 查询步骤结果

```http
GET /api/v1/case-runs/6001/step-results
Authorization: Bearer <token>
X-Workspace-Id: 1
```

响应语义：

- 返回真实执行后的线性步骤序列
- 若用例里含 `component_call`，其内部步骤会在此处展开

## 7. 常见错误与排查

### 7.1 后端无法启动

常见原因：

- PostgreSQL 未启动
- `AutoTestDev` 尚未创建
- Alembic 迁移未执行

建议排查顺序：

1. 确认 PostgreSQL 可连接
2. 执行 `python -m app.db.bootstrap`
3. 再次启动 `uvicorn app.main:app --reload`

### 7.2 登录失败

若默认管理员登录失败，优先排查：

- 初始化命令是否执行成功
- 当前连接的是否为 `AutoTestDev`
- 开发库是否被重建但未重新种子管理员

### 7.3 执行批次创建失败

#### 套件未激活

```json
{
  "error": {
    "code": "TEST_SUITE_NOT_ACTIVE",
    "message": "Test suite must be active before execution.",
    "details": []
  },
  "meta": {
    "request_id": "req_xxx"
  }
}
```

#### 存在未发布用例或组件

```json
{
  "error": {
    "code": "PUBLISHED_VERSION_REQUIRED",
    "message": "Component must be published before execution.",
    "details": []
  },
  "meta": {
    "request_id": "req_xxx"
  }
}
```

#### 套件为空

```json
{
  "error": {
    "code": "TEST_SUITE_EMPTY",
    "message": "Test suite must contain at least one test case before execution.",
    "details": []
  },
  "meta": {
    "request_id": "req_xxx"
  }
}
```

## 8. 联调口径说明

- 前端本地联调只关注开发库 `AutoTestDev`
- 测试数据库为后续测试阶段独立提供，不纳入当前前端联调流程
- 本次数据库切换属于基础设施升级，不改变现有 API 契约和业务响应结构
