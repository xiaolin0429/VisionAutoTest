# 接口示例与错误处理

## 1. 后端初始化示例

### 1.1 本地开发初始化

```bash
cd backend
source .venv/bin/activate
python -m app.db.bootstrap
```

### 1.2 启动后端

```bash
uvicorn app.main:app --reload
```

### 1.3 健康检查

```http
GET /healthz
```

## 2. 登录

### 2.1 请求

```http
POST /api/v1/sessions
Content-Type: application/json

{
  "username": "admin",
  "password": "admin123456"
}
```

说明：

- 上述 `password` 仅为示例值，实际联调请以本地 `backend/.env` 中 `VAT_DEFAULT_ADMIN_PASSWORD` 为准

## 3. 模板工作台主链路示例

### 3.1 上传媒体对象

```http
POST /api/v1/media-objects
Authorization: Bearer <token>
X-Workspace-Id: 1
Content-Type: multipart/form-data
```

表单字段：

- `file`
- `usage`
- `remark`

### 3.2 创建模板

```http
POST /api/v1/templates
Authorization: Bearer <token>
X-Workspace-Id: 1
Content-Type: application/json

{
  "template_code": "tpl_login_page",
  "template_name": "登录页模板",
  "template_type": "page",
  "match_strategy": "template",
  "threshold_value": 0.95,
  "status": "published",
  "original_media_object_id": 5001
}
```

说明：

- 创建模板时会自动生成首个基准版本
- 前端随后应读取模板详情中的 `current_baseline_revision_id`

### 3.4 触发 OCR 分析

```http
POST /api/v1/templates/6001/baseline-revisions/7001/ocr-results
Authorization: Bearer <token>
X-Workspace-Id: 1
```

说明：

- 请求体为空
- 成功后直接返回最新 OCR 快照
- 重复触发时覆盖更新原快照，不新增历史列表

### 3.5 从 OCR 结果生成 Mask 草稿并预览

```http
POST /api/v1/templates/6001/baseline-revisions/7001/preview-images
Authorization: Bearer <token>
X-Workspace-Id: 1
Content-Type: application/json

{
  "mask_regions": [
    {
      "name": "dynamic_title",
      "x_ratio": 0.1,
      "y_ratio": 0.1,
      "width_ratio": 0.35,
      "height_ratio": 0.22,
      "sort_order": 1
    }
  ]
}
```

成功示例：

```json
{
  "data": {
    "template_id": 6001,
    "baseline_revision_id": 7001,
    "source_media_object_id": 5001,
    "image_width": 200,
    "image_height": 100,
    "overlay_media_object_id": 8101,
    "overlay_content_url": "/api/v1/media-objects/8101/content",
    "processed_media_object_id": 8102,
    "processed_content_url": "/api/v1/media-objects/8102/content",
    "mask_regions": [
      {
        "name": "dynamic_title",
        "x_ratio": 0.1,
        "y_ratio": 0.1,
        "width_ratio": 0.35,
        "height_ratio": 0.22,
        "sort_order": 1
      }
    ]
  },
  "meta": {
    "request_id": "req_xxx"
  }
}
```

说明：

- 这是草稿预览，不会写回正式 `mask-regions`
- 前端应直接使用 `overlay_content_url` 和 `processed_content_url`
- `processed_content_url` 返回后端生成的玻璃态 Mask 展示图，接口字段不变；验收时应确认 Mask 区域不是黑色实心块，且底图内容仍可辨认。

### 3.6 使用正式 Mask 预览

```http
POST /api/v1/templates/6001/baseline-revisions/7001/preview-images
Authorization: Bearer <token>
X-Workspace-Id: 1
Content-Type: application/json

{}
```

说明：

- 空请求体或不传 `mask_regions` 时，后端使用模板当前已保存的正式 Mask

## 4. 错误处理

### 4.1 模板工作台常见错误码

| 错误码 | 含义 | 前端处理建议 |
|---|---|---|
| `BASELINE_REVISION_NOT_FOUND` | 基准版本不存在或不属于当前模板 | 回退当前面板状态并提示版本上下文失效 |
| `TEMPLATE_OCR_RESULT_NOT_FOUND` | 当前版本还没有 OCR 快照 | 当前通过 `GET 200 + status=not_generated` 返回，前端展示未生成态，不弹错误弹窗 |
| `TEMPLATE_OCR_ANALYSIS_FAILED` | OCR 本次执行失败 | 展示失败提示，可允许用户重试 |
| `TEMPLATE_PREVIEW_GENERATION_FAILED` | 预览图生成失败 | 展示失败提示，保留当前草稿 |
| `MASK_REGION_OUT_OF_RANGE` | Mask 越界 | 在编辑器上提示坐标非法 |
| `MEDIA_OBJECT_NOT_FOUND` | 媒体对象不存在 | 提示资源失效并建议刷新 |

### 4.2 OCR 失败响应示例

```json
{
  "error": {
    "code": "TEMPLATE_OCR_ANALYSIS_FAILED",
    "message": "Template OCR analysis failed.",
    "details": [
      {
        "field": "baseline_revision_id",
        "reason": "RuntimeError: ocr engine unavailable"
      }
    ]
  },
  "meta": {
    "request_id": "req_xxx"
  }
}
```

说明：

- 本次 `POST` 失败后，后端仍可能已写入失败态 OCR 快照
- 前端后续重新进入该版本时，`GET ocr-results` 可能返回 `status=failed`

### 4.3 预览失败响应示例

```json
{
  "error": {
    "code": "TEMPLATE_PREVIEW_GENERATION_FAILED",
    "message": "Template preview generation failed.",
    "details": [
      {
        "field": "baseline_revision_id",
        "reason": "RuntimeError: Failed to decode image bytes."
      }
    ]
  },
  "meta": {
    "request_id": "req_xxx"
  }
}
```

## 5. 联调建议

前端每次联调模板工作台时，建议固定抓三类数据：

1. 当前 `template_id`
2. 当前 `baseline_revision_id`
3. 当前 `X-Request-Id`

这样排查跨版本 OCR 结果串用、预览图错图和权限问题会更直接。
