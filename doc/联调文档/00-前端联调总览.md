# 前端联调总览

## 1. 适用范围

本文档面向当前 `VisionAutoTest` 前端与后端联调阶段，覆盖两类信息：

- 业务联调基线：认证、工作空间、编排、执行查询
- 基础设施变更：后端数据库已从默认 `SQLite` 切换为 `PostgreSQL + Alembic`

本文档以当前代码实现和最新数据库文档为准，不再沿用旧的 SQLite 联调口径。

## 2. 当前联调基线

- 后端运行基线：`FastAPI + SQLAlchemy + PostgreSQL + Alembic + BackgroundTasks`
- 接口前缀：`/api/v1`
- 健康检查：`GET /healthz`
- 默认管理员账号：`admin`，初始密码以本地 `backend/.env` 中的 `VAT_DEFAULT_ADMIN_PASSWORD` 为准
- 当前真实浏览器执行支持步骤：
  - `wait`
  - `click`
  - `input`
- 当前已支持 `component_call` 在执行时展开为真实步骤序列

## 3. 数据库与环境约定

### 3.1 开发联调数据库

前后端本地联调默认使用 PostgreSQL 开发库：

- 开发库：`AutoTestDev`
- 管理库：`postgres`
- 默认业务库连接：由本地 `backend/.env` 中的 `VAT_DATABASE_URL` 提供，示例形态如下：

```text
postgresql+psycopg://<db-user>:<db-password>@127.0.0.1:5432/AutoTestDev
```

### 3.2 初始化方式

后端在联调前应先完成数据库初始化：

```bash
cd backend
cp .env.example .env
# 填写本地数据库连接与管理员初始密码
python -m app.db.bootstrap
```

该命令会执行：

1. 若 `AutoTestDev` 不存在则创建数据库
2. 通过 Alembic 升级到最新迁移版本
3. 初始化默认管理员

### 3.3 测试数据库说明

- `AutoTestDev` 仅用于本地开发与自测
- 测试阶段使用独立测试数据库
- 前端联调不依赖测试数据库，也不应连接测试库

## 4. 最近变更对前端的影响

### 4.1 联调前提发生变化

以前的 SQLite 文件库不再是当前默认开发路径，前端本地联调需要确认后端：

- PostgreSQL 已启动
- `AutoTestDev` 已初始化
- Alembic 已迁移到最新版本

若未完成初始化，后端可能出现以下联调阻塞：

- 启动时报数据库不存在
- 启动后接口报表不存在或字段缺失
- 默认管理员无法登录

### 4.2 API 契约不因数据库切换而变化

本次数据库迁移不改变以下内容：

- 接口路径
- 通用响应结构
- 工作空间请求头规则
- 组件、用例、套件、执行链路的业务语义

前端无需因为 PostgreSQL 切换而修改接口协议层代码，但需要更新本地联调准备流程。

### 4.3 编排与执行规则继续生效

当前仍沿用上一轮执行链路增强规则：

- 组件状态：`draft / published / archived`
- 用例状态：`draft / published / archived`
- 套件状态：`draft / active / archived`
- 创建执行前：
  - 套件必须 `active`
  - 用例必须 `published`
  - 被引用组件必须 `published`
- `component_call` 在执行时会展开为真实步骤序列

## 5. 推荐联调流程

### 5.1 后端准备

建议前端在联调前确认后端已完成：

1. PostgreSQL 启动
2. 执行 `python -m app.db.bootstrap`
3. 启动后端服务 `uvicorn app.main:app --reload`
4. 调用 `GET /healthz` 返回成功

### 5.2 前端联调顺序

1. 登录获取令牌
2. 选择或创建工作空间
3. 创建环境档案
4. 创建组件与组件步骤
5. 创建测试用例与用例步骤
6. 创建测试套件并绑定用例
7. 调整资源状态到可执行态
8. 创建执行批次并轮询结果

## 6. 前端界面建议

### 6.1 本地环境提示

建议在开发环境 README、联调说明或本地调试页中提醒：

- 当前后端默认依赖 PostgreSQL，不再默认使用 SQLite 文件库
- 若后端尚未初始化数据库，应先执行 `python -m app.db.bootstrap`

### 6.2 执行入口提示

- 套件不是 `active` 时，禁用“立即执行”
- 用例不是 `published` 时，提示不可执行
- 组件不是 `published` 时，在 `component_call` 步骤附近提示未发布状态

### 6.3 执行详情展示

- `case-runs` 展示用例级执行结果
- `step-results` 展示真实执行步骤
- 对包含组件调用的场景，前端可先按线性列表展示，不强依赖组件树映射

## 7. 当前不在联调范围内

以下内容本轮后端未扩展，前端无需按已完成能力联调：

- `template_assert`
- `ocr_assert`
- 组件内部再次嵌套 `component_call`
- JWT 正式鉴权
- Celery + Redis 异步执行集群
- MySQL 运行时兼容验证
