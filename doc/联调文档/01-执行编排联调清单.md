# 执行编排联调清单

## 1. 联调准备清单

### 1.1 后端环境

| 项目 | 要求 |
|---|---|
| PostgreSQL | 已启动，`127.0.0.1:5432` |
| 开发库 | `AutoTestDev` |
| 初始化命令 | `python -m app.db.bootstrap` 已执行 |
| 后端服务 | 已启动并可访问 `/healthz` |
| Playwright | `playwright install chromium` 已执行 |
| 视觉依赖 | `opencv-python-headless`、`paddleocr`、`paddlepaddle` 已安装 |

### 1.2 请求头

| Header | 是否必填 | 说明 |
|---|---|---|
| `Authorization: Bearer <token>` | 是 | 除登录外都应携带 |
| `X-Workspace-Id` | 工作空间接口必填 | 工作空间内资源隔离 |
| `Idempotency-Key` | 建议 | 创建执行批次时建议传入 |
| `X-Request-Id` | 建议 | 便于联调排查 |

## 4. 编排载荷约束

### 4.1 状态枚举

| 资源 | 允许状态 |
|---|---|
| 组件 | `draft` / `published` / `archived` |
| 用例 | `draft` / `published` / `archived` |
| 套件 | `draft` / `active` / `archived` |

### 4.2 组件步骤

组件步骤当前支持：

- `wait`
- `click`
- `input`
- `template_assert`
- `ocr_assert`

步骤载荷约定：

- `wait`
  - `payload_json.ms`
- `click`
  - `payload_json.selector`
- `input`
  - `payload_json.selector`
  - `payload_json.text`
- `template_assert`
  - `template_id`
  - 可选 `payload_json.threshold`
- `ocr_assert`
  - `payload_json.selector`
  - `payload_json.expected_text`
  - 可选 `payload_json.match_mode`
  - 可选 `payload_json.case_sensitive`

### 4.3 用例步骤

用例步骤当前支持：

- `wait`
- `click`
- `input`
- `component_call`
- `template_assert`
- `ocr_assert`

其中 `component_call` 的联调要求：

- 必须传 `component_id`
- 执行时不会直接成为浏览器动作
- 后端会在运行时展开为组件内部真实步骤

### 4.4 模板与视觉规则

- `template_assert`
  - 必须传 `template_id`
  - 模板必须属于当前工作空间
  - 模板必须存在当前基准版本
  - 模板 `match_strategy` 必须为 `template`
- `ocr_assert`
  - 必须传 `payload_json.selector`
  - 必须传 `payload_json.expected_text`
  - 若关联模板，则模板 `match_strategy` 必须为 `ocr`

## 5. 前端联调检查点

### 5.1 创建执行前

| 检查项 | 合法值 |
|---|---|
| 套件状态 | `active` |
| 用例状态 | `published` |
| 被调用组件状态 | `published` |
| `template_assert` 模板策略 | `template` |
| `ocr_assert` 模板策略（若传模板） | `ocr` |
| 模板基准版本 | 必须存在 |

### 5.2 执行详情页

前端展示时需区分：

- `case-runs`：用例级结果
- `step-results`：真实执行步骤

若单个用例中存在 `component_call`：

- `step-results` 条数可能增加
- `step_no` 为真实执行顺序
- 当前接口不额外返回组件树结构

### 5.3 视觉断言证据链

`template_assert` 成功或失败时，前端应关注：

- `expected_media_object_id`
- `actual_media_object_id`
- `diff_media_object_id`（失败时通常存在）
- `score_value`

`ocr_assert` 成功或失败时，前端应关注：

- `actual_media_object_id`
- `score_value`
- `error_message`

### 5.4 状态语义

| 状态 | 含义 | 前端建议 |
|---|---|---|
| `passed` | 执行或断言通过 | 正常成功态 |
| `failed` | 业务断言不通过 | 展示差异或 OCR 不匹配信息 |
| `error` | 载荷/依赖/运行时异常 | 展示错误信息，避免误标为断言失败 |

## 6. 推荐错误码处理

| 错误码 | HTTP 状态 | 前端处理建议 |
|---|---|---|
| `TEST_SUITE_NOT_ACTIVE` | `422` | 提示“套件未激活，无法执行” |
| `PUBLISHED_VERSION_REQUIRED` | `422` | 提示“存在未发布的用例或组件，请先发布” |
| `TEST_SUITE_EMPTY` | `422` | 提示“套件为空，无法执行” |
| `STEP_SEQUENCE_INVALID` | `422` | 提示“步骤配置不完整或顺序非法” |
| `STEP_CONFIGURATION_INVALID` | `422` | 提示“步骤与模板策略不兼容” |
| `BASELINE_REVISION_REQUIRED` | `422` | 提示“模板缺少当前基准版本” |
| `IDEMPOTENCY_KEY_CONFLICT` | `409` | 提示“请勿重复触发相同执行请求” |
| `WORKSPACE_ID_REQUIRED` | `400` | 检查工作空间上下文是否丢失 |
| `WORKSPACE_FORBIDDEN` | `403` | 提示当前用户无工作空间权限 |
| `UNAUTHORIZED` / `TOKEN_*` | `401` | 提示重新登录 |
