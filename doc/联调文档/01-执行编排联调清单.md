# 执行编排联调清单

## 1. 联调准备清单

### 1.1 后端环境

联调前后端应确认以下前置条件：

| 项目 | 要求 |
|---|---|
| PostgreSQL | 已启动，账号 `postgres`，端口 `5432` |
| 开发库 | `AutoTestDev` |
| 初始化命令 | `python -m app.db.bootstrap` 已执行 |
| 后端服务 | 已启动并可访问 `/healthz` |

### 1.2 请求头

| Header | 是否必填 | 说明 |
|---|---|---|
| `Authorization: Bearer <token>` | 是 | 除登录外都应携带 |
| `X-Workspace-Id` | 工作空间接口必填 | 工作空间内资源隔离 |
| `Idempotency-Key` | 建议 | 创建执行批次时建议传入 |
| `X-Request-Id` | 建议 | 便于联调排查 |

## 2. 核心资源

| 资源 | 说明 |
|---|---|
| `components` | 组件主资源 |
| `components/{id}/steps` | 组件步骤 |
| `test-cases` | 测试用例 |
| `test-cases/{id}/steps` | 用例步骤 |
| `test-suites` | 测试套件 |
| `test-suites/{id}/cases` | 套件用例顺序 |
| `test-runs` | 执行批次 |
| `case-runs` | 用例执行实例 |
| `step-results` | 步骤执行结果 |
| `reports` | 执行报告 |

## 4. 编排载荷约束

### 4.1 状态枚举

| 资源 | 允许状态 |
|---|---|
| 组件 | `draft` / `published` / `archived` |
| 用例 | `draft` / `published` / `archived` |
| 套件 | `draft` / `active` / `archived` |

### 4.2 组件步骤

当前真实可执行步骤载荷约定：

- `wait`：`payload_json.ms`
- `click`：`payload_json.selector`
- `input`：`payload_json.selector`、`payload_json.text`

### 4.3 用例步骤

当前用例步骤支持：

- `wait`
- `click`
- `input`
- `component_call`

其中 `component_call` 的联调要求：

- 必须传 `component_id`
- 执行时不会直接成为浏览器动作
- 后端会在运行时展开为组件内部步骤

## 5. 前端联调检查点

### 5.1 创建执行前

| 检查项 | 合法值 |
|---|---|
| 套件状态 | `active` |
| 用例状态 | `published` |
| 组件状态 | `published` |

### 5.2 执行详情页

前端展示时需区分：

- `case-runs`：用例级结果
- `step-results`：真实执行步骤

若单个用例中存在 `component_call`：

- 步骤结果条数可能增加
- `step_no` 为真实执行顺序
- 当前接口不会额外返回步骤来源组件字段

### 5.3 数据库切换后的新增检查

若本地联调接口全部失败或登录失败，应优先检查：

1. PostgreSQL 是否启动
2. `AutoTestDev` 是否已初始化
3. 后端是否已执行 `python -m app.db.bootstrap`
4. 后端是否已完成 Alembic 迁移

## 6. 推荐错误码处理

| 错误码 | HTTP 状态 | 前端处理建议 |
|---|---|---|
| `TEST_SUITE_NOT_ACTIVE` | `422` | 提示“套件未激活，无法执行” |
| `PUBLISHED_VERSION_REQUIRED` | `422` | 提示“存在未发布的用例或组件，请先发布” |
| `TEST_SUITE_EMPTY` | `422` | 提示“套件为空，无法执行” |
| `IDEMPOTENCY_KEY_CONFLICT` | `409` | 提示“请勿重复触发相同执行请求” |
| `WORKSPACE_ID_REQUIRED` | `400` | 检查工作空间上下文是否丢失 |
| `WORKSPACE_FORBIDDEN` | `403` | 提示当前用户无工作空间权限 |
| `UNAUTHORIZED` / `TOKEN_*` | `401` | 提示重新登录，排查数据库初始化是否导致默认管理员缺失 |
