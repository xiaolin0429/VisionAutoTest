# 通知与集成模型

## 1. 表清单

| 表名 | 说明 |
|---|---|
| `notify_notification_channels` | 通知渠道表 |
| `notify_alert_rules` | 告警规则表 |
| `integ_webhook_endpoints` | Webhook 终端表 |
| `integ_webhook_deliveries` | Webhook 投递记录表 |

## 2. 表设计

### 2.1 notify_notification_channels

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK | 主键 |
| `workspace_id` | BIGINT | NOT NULL, FK | 工作空间 ID |
| `channel_name` | VARCHAR(128) | NOT NULL | 渠道名称 |
| `channel_type` | VARCHAR(32) | NOT NULL | `feishu/dingtalk/wecom/slack/email` |
| `endpoint_ciphertext` | TEXT | NOT NULL | 加密后的目标地址或凭据 |
| `status` | VARCHAR(32) | NOT NULL | 状态 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |
| `version_no` | INTEGER | NOT NULL DEFAULT 1 | 乐观锁 |
| `is_deleted` | BOOLEAN | NOT NULL DEFAULT FALSE | 逻辑删除 |

### 2.2 notify_alert_rules

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK | 主键 |
| `workspace_id` | BIGINT | NOT NULL, FK | 工作空间 ID |
| `rule_name` | VARCHAR(128) | NOT NULL | 规则名称 |
| `event_type` | VARCHAR(64) | NOT NULL | 事件类型 |
| `condition_json` | JSON | NOT NULL | 条件表达式 |
| `notification_channel_id` | BIGINT | NOT NULL, FK | 通知渠道 ID |
| `status` | VARCHAR(32) | NOT NULL | 状态 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |
| `version_no` | INTEGER | NOT NULL DEFAULT 1 | 乐观锁 |
| `is_deleted` | BOOLEAN | NOT NULL DEFAULT FALSE | 逻辑删除 |

### 2.3 integ_webhook_endpoints

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK | 主键 |
| `workspace_id` | BIGINT | NOT NULL, FK | 工作空间 ID |
| `endpoint_name` | VARCHAR(128) | NOT NULL | 终端名称 |
| `target_url` | VARCHAR(512) | NOT NULL | 回调地址 |
| `signing_secret_ciphertext` | TEXT | NULL | 加密签名密钥 |
| `retry_policy_json` | JSON | NOT NULL | 重试策略 |
| `status` | VARCHAR(32) | NOT NULL | 状态 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |
| `version_no` | INTEGER | NOT NULL DEFAULT 1 | 乐观锁 |
| `is_deleted` | BOOLEAN | NOT NULL DEFAULT FALSE | 逻辑删除 |

### 2.4 integ_webhook_deliveries

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK | 主键 |
| `webhook_endpoint_id` | BIGINT | NOT NULL, FK | Webhook 终端 ID |
| `event_type` | VARCHAR(64) | NOT NULL | 事件类型 |
| `request_body_json` | JSON | NOT NULL | 请求体快照 |
| `response_status_code` | INTEGER | NULL | 响应状态码 |
| `response_body_text` | TEXT | NULL | 响应体 |
| `delivery_status` | VARCHAR(32) | NOT NULL | `pending/success/failed` |
| `retry_count` | INTEGER | NOT NULL DEFAULT 0 | 重试次数 |
| `sent_at` | TIMESTAMP | NULL | 发送时间 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |

索引建议：

- `idx_integ_webhook_deliveries_endpoint_status(webhook_endpoint_id, delivery_status)`
- `idx_integ_webhook_deliveries_created_at(created_at)`

## 3. 设计说明

- 通知渠道和 Webhook 终端分离，便于区分“平台内通知”和“对外系统集成”。
- 投递记录为审计流水，禁止更新和删除。
- 密钥、地址等敏感信息统一按密文存储。
