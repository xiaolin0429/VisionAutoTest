# 媒体对象与引用模型

## 1. 表清单

| 表名 | 说明 |
|---|---|
| `asset_media_objects` | 媒体对象表 |

## 2. 表设计

### 2.1 asset_media_objects

用途：统一存储截图、基准图、热力图、录屏等媒体对象元数据。

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK | 主键 |
| `workspace_id` | BIGINT | NOT NULL, FK | 工作空间 ID |
| `storage_type` | VARCHAR(32) | NOT NULL | `local/s3/minio` |
| `bucket_name` | VARCHAR(128) | NULL | 存储桶 |
| `object_key` | VARCHAR(255) | NOT NULL | 对象键 |
| `file_name` | VARCHAR(255) | NOT NULL | 原文件名 |
| `mime_type` | VARCHAR(128) | NOT NULL | MIME 类型 |
| `file_size` | BIGINT | NOT NULL | 文件大小 |
| `checksum_sha256` | VARCHAR(64) | NOT NULL | 摘要 |
| `status` | VARCHAR(32) | NOT NULL | `active/archived/deleted` |
| `usage` | VARCHAR(32) | NOT NULL | `template_source/baseline/artifact` |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `created_by` | BIGINT | NULL | 上传人 |

索引建议：

- 唯一约束：`uk_asset_media_objects_checksum(checksum_sha256, file_size, workspace_id)`
- 索引：`idx_asset_media_objects_workspace_id(workspace_id)`
- 索引：`idx_asset_media_objects_usage_status(usage, status)`

## 3. 引用关系

- `asset_baseline_revisions.media_object_id`：引用基准图对象。
- `exec_step_results.expected_media_object_id`：引用预期图对象。
- `exec_step_results.actual_media_object_id`：引用实际图对象。
- `exec_step_results.diff_media_object_id`：引用差异图对象。
- `report_artifacts.media_object_id`：引用报告证据链对象。

## 4. 设计说明

- 媒体对象与业务对象解耦，便于未来切换对象存储。
- 媒体对象内容一经创建不可原地修改；如文件内容变化，必须创建新记录。
- 逻辑删除前必须校验引用关系；若仍被业务对象引用，不允许删除。
- 对外下载建议返回短时有效地址或对象键，而不是直接内联大文件。
