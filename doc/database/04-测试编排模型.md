# 测试编排模型

## 1. 表清单

| 表名 | 说明 |
|---|---|
| `case_components` | 公共组件主表 |
| `case_component_steps` | 公共组件步骤表 |
| `case_test_cases` | 测试用例主表 |
| `case_test_case_steps` | 测试用例步骤表 |
| `case_test_suites` | 测试套件主表 |
| `case_suite_cases` | 套件用例关联表 |

## 2. 表设计

### 2.1 case_components

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK | 主键 |
| `workspace_id` | BIGINT | NOT NULL, FK | 工作空间 ID |
| `component_code` | VARCHAR(64) | NOT NULL | 组件编码 |
| `component_name` | VARCHAR(128) | NOT NULL | 组件名称 |
| `status` | VARCHAR(32) | NOT NULL | `draft/published/archived` |
| `description` | VARCHAR(512) | NULL | 描述 |
| `published_at` | TIMESTAMP | NULL | 发布时间 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |
| `version_no` | INTEGER | NOT NULL DEFAULT 1 | 乐观锁 |
| `is_deleted` | BOOLEAN | NOT NULL DEFAULT FALSE | 逻辑删除 |

### 2.2 case_component_steps

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK | 主键 |
| `component_id` | BIGINT | NOT NULL, FK | 组件 ID |
| `step_no` | INTEGER | NOT NULL | 步骤序号 |
| `step_type` | VARCHAR(32) | NOT NULL | `click/input/assert/template_assert/ocr_assert` |
| `step_name` | VARCHAR(128) | NOT NULL | 步骤名称 |
| `template_id` | BIGINT | NULL, FK | 关联模板 |
| `payload_json` | JSON | NOT NULL | 步骤参数 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |
| `updated_at` | TIMESTAMP | NOT NULL | 更新时间 |

唯一约束：

- `uk_case_component_steps(component_id, step_no)`

| 字段 | 类型 | 约束 | 说明 |

### 2.5 case_test_suites

| 字段 | 类型 | 约束 | 说明 |

| `suite_name` | VARCHAR(128) | NOT NULL | 套件名称 |

### 2.6 case_suite_cases

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| `id` | BIGINT | PK | 主键 |
| `test_suite_id` | BIGINT | NOT NULL, FK | 套件 ID |
| `test_case_id` | BIGINT | NOT NULL, FK | 用例 ID |
| `sort_order` | INTEGER | NOT NULL | 执行顺序 |
| `created_at` | TIMESTAMP | NOT NULL | 创建时间 |

约束与索引：

- 唯一约束：`uk_case_suite_cases(test_suite_id, test_case_id)`
- 唯一约束：`uk_case_suite_sort(test_suite_id, sort_order)`

## 3. 设计说明

- 编排步骤采用明细表，避免整棵执行树直接固化为不可治理的大 JSON。
- `payload_json` 只承载步骤细节，不承载主关系。
- 套件和用例采用关联表，方便排序、复用和后续统计。
