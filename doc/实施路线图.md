# 实施路线图

## 第一阶段：契约冻结

- 冻结 API 资源模型、请求响应结构、错误码和分页规则。
- 冻结数据库逻辑模型、主外键、索引和状态枚举。
- 输出 OpenAPI 初版和 Alembic 初版迁移清单。

## 第二阶段：最小后端可运行

- 完成认证、工作空间、环境、模板、用例、执行、报告模块的基础 API。
- 完成数据库迁移和基础种子数据。
- 完成本地文件存储和最小异步任务机制。

## 第三阶段：视觉执行闭环

- 接入 Playwright 截图与驱动操作。
- 接入 OpenCV 模板匹配和忽略区域算法。
- 接入 OCR 文本识别。
- 形成报告差异图与失败证据链。

## 第四阶段：前端管理台

- 完成登录、工作空间、模板、用例、套件、执行、报告页面。
- 完成忽略区域绘制组件。
- 对接所有 MVP API。

## 第五阶段：企业级增强

- 引入 Redis + Celery。
- 扩展对象存储。
- 增加通知渠道、Webhook、告警规则。
- 增加监控、限流、审计与可观测性能力。

## 里程碑产物

- M1：OpenAPI 文档、数据模型文档、迁移脚本草案。
- M2：后端最小服务可启动，Swagger 可访问。
- M3：单条测试用例可执行并生成报告。
- M4：前端可完成完整业务操作闭环。
- M5：支持单机稳定运行和初级 CI 集成。
